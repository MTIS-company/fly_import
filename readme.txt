Все файлы проекта находятся в папке flytechnology (далее по тексту "рабочая папка")
В папке flytechnology/folders_img_parser находится парсер parse.php, который служит для создания списка картинок категорий товаров (отсутствующих в каталоге flytechnology). Парсер анализирует содержимое корневого раздела товаров на сайте flytechnology.ua, рекурсивно проходя все уровни вложенности без ограничений и создает файл folders_img.xml в текущей папке. Парсер работает с применением библиотеки phpQuery.
Если необходимо обновить сведения об изображениях категорий - выполняем парсер и переносим созданный xml файл в рабочую папку. Корневой раздел сайта поставщика изображений не содержит.

Чтобы приступить к импорту, выполняем скрипт index.php, находящийся по URL рабочей папки.
Скрипт предлагает пользователю ввести URI xml каталога поставщика и начинает импорт. Вначале импортируются все категории товаров, отсутствующие в каталоге navi. Идентификация принадлежности товарных категорий и товаров к каталогу flytechnology происходит по полю XML_ID, в которое скрипт добавляет символьный префикс. Символьные префиксы товаров и категорий прописаны в файле config.php (CATALOG_PREFIX=, ITEM_PREFIX=). Там же прописан ID корневой папки каталога navi (SECTION=), в которую будет выполняться импорт. Если указать пустое значение - импорт будет выполнен в корневой раздел каталога navi. Чтобы категории не затерялись после импорта, рекомендуется создать папку и прописать ее ID в config.php. Чтобы папка не высвечивалась в каталоге товаров на сайте, ее можно деактивировать. После импорта пользователю предоставляется доступ к списку импортированных папок с возможностью их редактирования. Папки можно перемещать, что не приведет к нарушению иеррархии при последующем импорте.

В каталоге поставщика товары привязаны к категориям по полю <имя категории>. Поэтому скрипт выполняет нормализацию таблицы, привязывая товар к категории по ID категории. Т.о. даже при изменении наименования категории в каталоге поставщика иеррархия объектов не нарушается. Импорт товаров выполняется скриптом import_products.php ajax вызовом по 5 товаров. Следующие 5 товаров начинают импортироваться только после завершения импорта предыдущих 5-ти с целью снижения нагрузки на сервер. Импортируются только товары, отсутствующие в каталоге navi. После завершения импорта пользователь получает доступ к списку импортированных товаров с возможностью их редактирования. А также возможность деактивации товаров, которые есть в каталоге navi, но отсутствуют в каталоге поставщика и возможность активации товаров, неактивных в каталоге navi, но присутствующих в каталоге поставщика. Эти функции выполняет скрипт deactivate_products.php.

Также в рабочей папке содержится файл manual.txt, содержащий руководство по разертыванию bitrix backup на localhost.
